# 数据库配置指南

为确保项目在新环境中正常连接 MySQL，请按以下步骤填写必要的配置：

## 1. 核心配置文件

**文件路径：** `backend/application.properties`

需要填写的键值：
- `spring.datasource.url` – 目标数据库的 JDBC 地址，例如：`jdbc:mysql://<host>:3306/<database>?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true`
- `spring.datasource.username` – 数据库用户名
- `spring.datasource.password` – 数据库密码

> 说明：该文件在默认情况下启用 `mysql` Profile，应用启动时会优先读取这些值。

## 2. Profile 专用配置

**文件路径：** `backend/src/main/resources/application-mysql.properties`

在如下键位中填写与上一步相同或针对特定环境的值：
- `spring.datasource.url`
- `spring.datasource.username`
- `spring.datasource.password`

若不同环境需要不同的连接信息，可只在此文件中填写，并保持 `backend/application.properties` 中的值为空，让 Profile 接管。

## 3. 可选：JDBC 测试类

**文件路径：**
- `backend/src/main/java/com/example/studentanalysissystem/test/JdbcDirectTest.java`
- `backend/src/main/java/com/example/studentanalysissystem/test/SimpleJdbcTest.java`

这些调试类中含有示例用的 JDBC URL、用户名和密码。如果需要在新环境运行这些类，请同步修改其中的连接字符串。

## 4. 初始化数据库（可选）

**脚本：** `database/newsql.sql`

如需快速初始化数据库结构，可在目标环境执行：
```
mysql -u <username> -p < database/newsql.sql
```

## 5. 重启服务

更新配置后，请重启后端服务，以确保新的数据库连接信息生效。

> 建议将真实的数据库凭据保存在环境变量或密钥管理服务中，避免以明文形式提交到版本控制系统。
